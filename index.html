Final Project - SCHOOL SIMULATOR 2025

Motivation of the project:

This project was taken on as it was capable of making us use all the pillars of OOP and helped me learn how to work with databases, as this was my first time working with SQL. These SQL skills will be helpful for my upcoming research program as I will need to create a database for all the data I will be collecting.

Functional requirements / what the project does:

This project simulates a school database. On the website, a user will be able to add a person to the database (person, student, alumni, employee, faculty) and there are a variety of database queries based on what information a user may have of a person. In addition to person queries, there are also department, building, and room queries if you don't have any person information. Ex: if a user only knows about the engineering department, but not any staff there, they can query by department and get a list of all people who are associated with said department. The list will have all the people's names and contact information.

Project design / Implementation:

The primary component that I developed was the service and table classes. These classes are crucial because they provide the functionality that enables users to perform queries on the database. Essentially, the service classes ensure that the data retrieval and manipulation processes are executed correctly. The table classes represent the database tables and include methods for interacting with the data stored in these tables. Together, these classes form the core of the system that allows users to efficiently access and manage the database contents.
